FLOW_MAP
[FLOW-001] App boot and initial data load
- Entry: frontend-src/js/app/components/App.jsx -> AppShell + ProjectsModule.
- ProjectsModule mount: frontend-src/js/app/modules/projects/projects.js.
- Load backend public config: apiFetchBackendConfig() -> GET /public-config.
- Load projects: apiFetchProjects() -> GET /projects.

[FLOW-002] AJAX request lifecycle (frontend)
- Config bootstrap: frontend-src/js/app/services/api.js::initApiConfig() loads frontend-public/app-public-config.json.
- Request builder: apiRequest(apiRoute, data, method, options).
- Error contract: throws Error when response.ok is false OR payload.success !== true.
- Error shape on throw: error.status, error.payload, error.errors.

[FLOW-003] Backend request lifecycle
- Entry: backend/index.php -> backend/api.php.
- Bootstrap: load_config(), handle_cors(), include shared classes, include module entry files.
- Router: backend/routes.php creates AltoRouter and auto-loads backend/modules/*/routes.php.
- Dispatch: matched target callback executes handler, then JSON is returned.

[FLOW-004] Create project end-to-end
- FE submit: CreateProjectForm -> formFn().handleSubmit() -> apiCreateProject().
- API call: POST /projects/add.
- Handler: backend/modules/projects-manager/handlers.php::CreateProjectCb().
- Service: backend/modules/projects-manager/ProjectsManager.php::tryCreateProject().
- Response branch A: 201 success with project payload.
- Response branch B: 201 success plus issues in errors field (partial warnings).
- Response branch C: 422 fail via sendErrorResponse() with keyed validation/runtime errors.

KNOWN_LOGIC_GAPS_FOR_AI
- KG-001 status naming mismatch:
  - Backend create flow sets statuses like has_warnings/has_errors in ProjectsManager::prepareProjectStatus().
  - ProjectCard visual mapping expects active/error/warning/inactive in ProjectCard.jsx::statusClass.
  - Impact: warning/error indicator styling can be inconsistent for newly created projects with warnings.
- KG-002 notification API call shape mismatch:
  - notifications.js::showNotification(message, type, autoHideDelay, options).
  - CreateProjectForm calls showNotification(message, type, { ...options }) with options object as third argument.
  - Impact: auto-hide timing/options can behave unexpectedly.
- KG-003 forms service import drift in CreateProjectForm:
  - CreateProjectForm imports normalizeBackendIssues from forms.js, but normalized export is currently not active.
  - Impact: build/runtime import issues depending on bundler strictness.
